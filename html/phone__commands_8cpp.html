◆ readPhoneCommand()

void readPhoneCommand ( uint8_t conf_data[MAX_MSG_LEN_PHONE])

Config Messages length 1B - Msg ID 1B - Data Msg ID: 0x10 - Hello Message (followed by 0x20, 0x30) 0x20 - Timestamp from phone 0x50 - Callsign 0x55 - Wifi SSID and PW 0x70 - Latitude 0x80 - Longitude 0x90 - Altitude 0x95 - APRS Symbols 0xA0 - Textmessage 0xF0 - Save Settings to Flash Data: Callsign: length Callsign 1B - Callsign Latitude: 4B Float Longitude: 4B Float Altitude: 4B Integer

WiFi SSID and PWD: 1B - SSID Length - SSID - 1B PWD Length - PWD

Position Settings from phone are: length 1B | Msg ID 1B | 4B lat/lon/alt | 1B save_settings_flag Save_flag is 0x0A for save and 0x0B for don't save If phone send periodicaly position, we don't save them.

currently we save the settings when the last config arrives which is APRS SYMBOLS - adapt is needed!
